# نصب پنل روی سرور ایران از طریق تانل SSH

اسکریپت‌هایی برای دور زدن فیلترینگ هنگام نصب پنل‌های مدیریت سرور (مثل HestiaCP) روی سرورهای داخل ایران. یک تانل SSH موقت به یک سرور خارج از ایران برقرار می‌شود، ترافیک نصب (apt/curl/git) از آن عبور می‌کند، و در پایان با یک اسکریپت جدا کاملاً پاک می‌شود.

## پیش‌نیازها

- یک سرور/VPS خارج از ایران با دسترسی SSH (به‌عنوان relay) که اینترنت آزاد دارد.
- یوزر/پسورد SSH همان سرور خارج (اتصال با رمز عبور انجام می‌شود، نه کلید).
- سرور ایران با دسترسی روت، بر پایه Debian/Ubuntu.

## نحوه کار

```
سرور ایران  --SSH tunnel (SOCKS5)-->  سرور خارج  --> اینترنت آزاد
     |
     +-- privoxy (تبدیل SOCKS5 به HTTP روی 127.0.0.1:8118)
     |
     +-- apt / curl از این پروکسی محلی استفاده می‌کنند
```

1. `install-hestia-via-tunnel.sh` تانل SSH با پورت‌فورواردینگ داینامیک (`-D`) به سرور خارج برقرار می‌کند.
2. `privoxy` نصب و طوری تنظیم می‌شود که SOCKS5 تانل را به یک پروکسی HTTP محلی تبدیل کند (چون `apt-get` از SOCKS5 پشتیبانی نمی‌کند).
3. اتصال از طریق پروکسی تست می‌شود، سپس نصب‌کننده رسمی HestiaCP دانلود و با همان پروکسی اجرا می‌شود.
4. بعد از اطمینان از سالم بودن پنل، `remove-tunnel.sh` تانل، تنظیمات apt و privoxy را کامل حذف می‌کند.

## استفاده

### ۱) اجرا روی سرور ایران

نیازی به ادیت کردن فایل نیست؛ اسکریپت مقادیر لازم را در ترمینال می‌پرسد.

```bash
scp install-hestia-via-tunnel.sh remove-tunnel.sh root@IRAN_SERVER:/root/
ssh root@IRAN_SERVER
chmod +x install-hestia-via-tunnel.sh remove-tunnel.sh
./install-hestia-via-tunnel.sh
```

هنگام اجرا این مقادیر پرسیده می‌شود:

- آدرس IP یا دامنه سرور خارج (relay)
- پورت SSH سرور خارج (پیش‌فرض `22`)
- یوزر SSH سرور خارج (پیش‌فرض `root`)
- رمز عبور SSH سرور خارج (در ترمینال نمایش داده نمی‌شود)
- هاست‌نیم و ایمیل ادمین پنل (اختیاری — با Enter خالی، نصب به‌صورت تعاملی می‌پرسد)

اتصال با کمک ابزار `sshpass` انجام می‌شود؛ اگر نصب نباشد، اسکریپت خودش آن را نصب می‌کند.

### ۲) پاک‌سازی بعد از اطمینان از سالم بودن نصب

```bash
./remove-tunnel.sh
```

این اسکریپت تانل SSH، فایل پروکسی apt (`/etc/apt/apt.conf.d/95-hestia-tunnel-proxy`)، تنظیمات اضافه‌شده به `privoxy` و خود بسته `privoxy` را حذف می‌کند.

## نکات امنیتی

- رمز عبور فقط در حافظه اسکریپت نگه‌داری می‌شود و بعد از برقراری تانل پاک (`unset`) می‌شود.
- پروکسی privoxy فقط روی `127.0.0.1` گوش می‌دهد و از بیرون قابل دسترس نیست.
- بعد از اتمام کار حتماً `remove-tunnel.sh` را اجرا کنید تا تانل باز نماند.
